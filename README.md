## วิธีการทำงานร่วมกันของทีม 

### 1. Clone และเตรียม Repository 
- สมาชิก clone Team Repository ลงเครื่องตนเอง 
- แต่ละคนสร้าง **branch ของตัวเอง** ตัวอย่าง: Pichamon_3954, Thitikorn_5998 

### 2. พัฒนาโค้ดบน Branch ของตนเอง 
- แต่ละคนแก้ไขเฉพาะไฟล์ที่ตนรับผิดชอบในโฟลเดอร์ Project/ 

### 3. Commit และ Push 
- Commit อย่างสม่ำเสมอ พร้อมเขียนข้อความให้ชัดเจน 

### 4. Pull Request / Merge 
1. สลับไป main 
2. Merge branch ของตัวเอง 
3. อีกคนรอให้ merge เสร็จ แล้วทำเหมือนกัน 
4. ถ้ามี conflict ให้แก้ร่วมกันในไฟล์ 

### 5. การตรวจสอบและทดสอบร่วมกัน 
1. หลัง merge แล้ว pull โค้ดล่าสุดลงเครื่อง 
2. รันโปรแกรมทดสอบผลลัพธ์ด้วย input หลายรูปแบบ 

## Team Workflow 
1. รับ Assignment → สร้าง Team Repository 
2. เพิ่ม Collaborators 
3. สร้างโครงสร้างโฟลเดอร์ (AboutMyTeam, Project) 
4. แต่ละคนสร้าง branch ของตัวเอง 
5. พัฒนา Bubble Sort และ Quick Sort แยกกัน 
6. ทดสอบแต่ละส่วน 
7. Merge เข้าสู่ main 
8. แก้ Conflict (ถ้ามี) 
9. ทดสอบโปรแกรมรวม 
10. ส่งงานผ่าน GitHub Classroom
